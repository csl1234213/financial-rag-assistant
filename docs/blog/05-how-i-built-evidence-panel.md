# How I Built the Evidence Panel in a Financial RAG System

# 我是如何在金融 RAG 系统中构建 Evidence Panel 的

**Version / 版本：V2.2 Stable**

---

## Introduction / 项目背景

In most RAG systems, the final output is a single answer generated by the LLM.

在大多数 RAG 系统中，最终输出通常只是大模型生成的一段回答。

At the beginning of this project, my system was no different.

在项目初期，我的系统也是如此。

It could retrieve relevant chunks and generate answers, but it had one major problem:

它可以检索相关内容并生成答案，但有一个核心问题：

> The user could not verify where the answer came from.

用户无法验证答案来源。

This is unacceptable in financial analysis scenarios.

在金融分析场景中，这是不可接受的。

Because financial decisions require traceability.

因为金融决策必须可追溯。

---

## The Problem / 核心问题

Initially, the system worked like this:

最初系统流程是：

```text id="k3p9lm"
Question
   ↓
Retriever
   ↓
LLM
   ↓
Answer
```

The answer was often correct, but opaque.

答案通常是正确的，但不可解释。

Users could not see:

用户无法看到：

* Which document was used
* Which paragraph was referenced
* Whether the answer was supported by evidence

This created a trust problem.

这带来了信任问题。

---

## First Attempt / 初步尝试

My first solution was simple:

最初的解决方案很简单：

> Attach retrieved chunks directly to the answer.

把检索到的 chunks 直接附加在答案后面。

This looked like:

表现形式类似：

```text id="q7m2vn"
Answer:
...
```

```text
Evidence:
- Chunk 1
- Chunk 2
- Chunk 3
```

However, this approach had issues:

但这个方法存在问题：

* Evidence was unstructured
* Repetition of irrelevant chunks
* Hard to read for users
* No clear mapping between answer and evidence

So it was not scalable.

因此不具备可扩展性。

---

## Design Decision / 设计决策

I decided to introduce a dedicated module:

于是我设计了一个独立模块：

> Evidence Panel

Evidence Panel is not just a UI component.

它不仅仅是一个 UI 组件。

It is a structured layer between retrieval and final output.

它是连接检索与最终输出的结构化中间层。

---

## Final Architecture / 最终架构

After refactoring, the system became:

重构后的系统如下：

```text id="p8c1tr"
Question
   ↓
Retriever
   ↓
Context Builder
   ↓
LLM
   ↓
Answer + Evidence Panel
```

Key change:

关键变化：

> Evidence is no longer an afterthought, but part of the system design.

证据不再是附加内容，而是系统设计的一部分。

---

## How Evidence Panel Works / 实现方式

The Evidence Panel is built from three layers:

Evidence Panel 由三层组成：

---

### 1. Chunk Selection / 证据筛选

The retriever returns top-k chunks:

检索器返回 top-k chunks：

* ranked by semantic similarity
* filtered for relevance
* grouped by source document

---

### 2. Context Structuring / 上下文结构化

Instead of passing raw chunks directly to the LLM,

不再直接把原始 chunks 传给模型，

a Context Builder formats them:

而是通过 Context Builder 进行整理：

* Deduplicate information
* Add document source metadata
* Maintain ordering by relevance

---

### 3. Evidence Formatting / 证据格式化

Finally, evidence is formatted into a structured panel:

最后将证据格式化为结构化展示：

* Source document name
* Chunk content
* Relevance ranking
* Highlighted text segments

---

## Why This Design Matters / 为什么这个设计重要

The key goal was not just to improve UI.

核心目标不仅仅是改善 UI。

It was to improve system transparency.

而是提升系统透明度。

In financial applications:

在金融应用中：

> An answer without evidence is not useful.

没有证据的答案是没有价值的。

Evidence Panel makes the system:

Evidence Panel 让系统变得：

* Auditable / 可审计
* Traceable / 可追溯
* Explainable / 可解释

---

## Challenges / 实现过程中的问题

### 1. Over-retrieval noise / 证据噪声问题

Initial retrieval returned too many irrelevant chunks.

最初检索会返回过多无关内容。

Solution:

解决方法：

* stricter top-k filtering
* similarity threshold tuning

---

### 2. Context overflow / 上下文过长

Too many chunks caused prompt overflow.

chunk过多导致上下文超长。

Solution:

解决方法：

* chunk compression
* context trimming
* prioritization based on similarity

---

### 3. Source duplication / 来源重复

Same document appeared multiple times.

同一文档重复出现。

Solution:

解决方法：

* deduplication layer in Context Builder

---

## Key Insight / 核心收获

The most important realization was:

最重要的认知是：

> In RAG systems, retrieval is not enough — evidence presentation is equally important.

在 RAG 系统中，检索本身是不够的，证据展示同样重要。

A system that cannot explain its output is not reliable in real-world scenarios.

无法解释输出的系统，在真实场景中是不可信的。

---

## Conclusion / 总结

The Evidence Panel transformed the system from:

Evidence Panel 让系统从：

> a question-answering tool

变成：

> a financial research assistant

It shifted the focus from:

也让系统的重点从：

* “generate answers”
  to
* “support reasoning with evidence”

从“生成答案”转变为“支持推理与证据展示”。

This is one of the most important architectural decisions in the project.

这是本项目中最重要的架构决策之一。
